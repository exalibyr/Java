Для работы приложения необходим JDK 8, а также локальный или удалённый сервер.
При изменения адреса сервера необходимо изменить его URL адрес.
Для этого нужно в файле nanobasemodule\src\main\java\logic\DataManager.java следующую константу:

private static final String URL = "jdbc:mysql://localhost:3306/nanobase"
Сдесь в качестве примера указан адрес в случае локального сервера. 
"jdbc:mysql:" - это строка драйвера базы данных. Она зависит от установленной СУБД.
"localhost:3306/nanobase" - адрес сервера. 
В случае перехода, например, с локального сервера на удалённый необходимо изменить вторую часть строки на соответствующий URL сервера и перекомпилировать исполняемый архив

Для работы в приложении необходимо авторизоваться в системе. Нужно ввести логин и пароль, которые хранятся в базе данных в таблице users.
Пароль в таблице users необходимо хранить в зашифрованном MD5 виде.
В поле пароля при авторизации в приложении вводится пароль в явном виде. Программа сама зашифрует пароль и проверит его.

Несколько примеров как вводить данные по свойстам композитов для корректного отображения:
1) Свойство в статье имеет значение 10 и единицы измерения см.
В поле ANSWER_TEXT таблицы nano_f_answers  вводится значение: 10.
В поле ANSWER_NAME таблицы nano_l_answers  вводится значение: см.
2)Свойство в статье имеет интервальное значение 10-15 и единицы измерения см.
В поле ANSWER_TEXT таблицы nano_f_answers  вводится значение: 10-15.
В поле ANSWER_NAME таблицы nano_l_answers  вводится значение: см.
3) Свойство в статье имеет несколько значений при различных условиях. 
Например 10 при 100 K, 20 при 200 К, 30 при 300 К и единицы измерения см.
В поле ANSWER_TEXT таблицы nano_f_answers  вводится значение: 10; 20; 30.
В поле ANSWER_NAME таблицы nano_l_answers  вводится значение: см; 100 K; 200 K; 300 K.
При таком вводе приложение распознает формат и сможет интерпретировать данные.
